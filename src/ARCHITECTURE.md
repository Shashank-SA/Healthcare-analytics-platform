# Enterprise Architecture Specification: Healthcare Analytics Platform
This document outlines the complete folder structure, file responsibilities, and architectural boundaries designed for a highly secure, HIPAA-compliant, and horizontally scalable Healthcare Analytics module integrated inside an enterprise Electronic Medical Record (EMR) system.

---

## Part 1: System Topology & Separation of Concerns

The Healthcare Analytics Platform is structured as a decoupled **two-tier architecture** with strict separation of the User Interface layer (Frontend) and the Analytics Service/Data layer (Backend). This guarantees that:
1. **Zero PHI Leakage:** Personal Health Information (PHI) is processed, masked, and query-audited server-side before any analytical data touches external model endpoints (e.g., Google Gemini API).
2. **Independent Scalability:** The Analytics Engine runs on a stateless backend cluster, meaning intensive aggregations, report building, and exporting can scale out horizontally without affecting core EMR transactions.
3. **High-Density Performance:** Frontend components leverage optimized caching (`React Query` / `Axios` interceptors) to render rich analytical graphs on-demand without blocking the main browser thread.

## Part 3: Folder & File Responsibilities

### 1. Frontend Responsibilities (Client Portal)

- **`src/components/analytics/charts/`**: Responsible for creating clean, reusable visual graphs. They wrap `Recharts` and support dynamic resizing, tooltips, responsive margins, and customized color themes (matching clinical profiles like Emerald, Pink, Violet, Indigo, and Amber). They accept ready-made datasets directly.
- **`src/components/analytics/filters/`**: Holds dropdown bars that handle custom selections. Changes trigger state modifications that dynamically reload the `Dashboard` or a specific report without full-page re-renders.
- **`src/components/analytics/tables/`**: Responsible for the **Cohort details** spreadsheet view. In a high-volume clinical workspace, it manages large arrays (100+ rows) smoothly with column alignments, medical tagging (ICD-10 classifications), and rapid sorting.
- **`src/pages/Dashboard/`**: Orchestrates the main workstation console view. It displays default KPI metrics, suggestions, and the prompt box.
- **`src/pages/GeneratedReport/`**: Responsible for displaying a fully-rendered report. It pulls RPT codes and builds cards, tables, charts, and the AI summary section.
- **`src/services/api/`**: Encapsulates external interactions. Uses `Axios` with a base URL, handling automatic headers (such as forwarding credentials or EMR authorization tokens) and formatting response payloads.

---

### 2. Backend Core Infrastructure

- **`server.ts` & `app.ts`**: The core application engine. Handles Express configuration, mounts compression libraries (Gzip), parses JSON payloads, and handles error boundaries safely.
- **`src/config/`**: Separates settings from operational logic. Isolates environment variables and ensures required keys (such as `DATABASE_URL` and `GEMINI_API_KEY`) are present and valid before startup.
- **`src/database/prisma/`**: Directs database synchronization. Manages DB models and relational links (between patients, encounters, medical codes, reports, history, and users) cleanly.

---

### 3. Analytics Core Modules (The 7 Pillars of Health Intelligence)

#### A. Analytics Engine (`src/services/analytics/analytics-engine/`)
- **Responsibility**: Calculates KPIs, statistics, aggregates demographic arrays, and determines trends over 30/90 days. It does not touch raw medical values directly; it queries repositories and transforms data arrays into statistical summaries.
- **Inputs**: Filter metrics (such as Date Range, Provider ID, Department).
- **Outputs**: Formatted KPI payloads (counts, percentage changes) and graphical datasets (monthly visits arrays, diagnosis percentages).

#### B. Prompt Parser (`src/services/analytics/prompt-parser/`)
- **Responsibility**: Interprets clinical intent from raw natural language prompts. It converts unstructured strings (e.g., *"Show patient visits today"*) into structured target configurations.
- **Integration**: Feeds the prompt to a dedicated Google Gemini model inside `prompt-parser/gemini/client.ts` using structured schema definitions to return a consistent JSON schema representing the intent (e.g., `queryType`, `dateRange`, `primaryEntity`, `aggregations`).
- **Safety**: Employs deep schema verification to ensure the model output strictly follows a structured JSON payload, preventing unexpected prompt injection or code-injection vectors.

#### C. Privacy Module (`src/services/analytics/privacy/`)
- **Responsibility**: The gateway for HIPAA compliance. Operates as a double-sided proxy filter.
- **Outbound Filtering**: Sanitizes the prompt before sending it to Google Gemini. If a user inputs *"Show visits for John Doe"*, it interacts with Microsoft Presidio (`privacy/presidio/client.ts`) to redact/mask patient names, MRNs, phone numbers, or dates, changing the prompt to a safe generic structure.
- **Inbound Filtering**: Audits the summaries or text returned from Gemini to ensure no accidental leaks or hallucinatory references to real PHI occur.

#### D. Query Builder (`src/services/analytics/query-builder/`)
- **Responsibility**: Translates parsed clinical intent into local database operations.
- **Strict Rule**: **Never allows Gemini to generate SQL directly.** Gemini only parses the clinical intent (as JSON). The Query Builder takes this structured intent (e.g., `{ entity: "encounter", filter: { date: "today" } }`) and programmatically builds PostgreSQL queries using safe Prisma ORM operations. This completely eliminates SQL injection risks.

#### E. Summary Service (`src/services/analytics/summary-service/`)
- **Responsibility**: Generates contextual medical summaries. Instead of sending raw patient databases to the AI, it collects sanitized, high-level aggregated results (e.g., *"Outpatient visits grew 8.6%, Amoxicillin remains high"*) and sends this statistical outline to Gemini to generate a polished, highly-professional medical summary paragraph.

#### F. Export Module (`src/utils/exports/`)
- **Responsibility**: Converts compiled reports into downloadable formats. Uses `pdfkit` to build styled PDFs (complete with clinical banners, KPI cards, and charts) and `exceljs` to render cohort details tables into functional spreadsheets.

#### G. History Module (`src/services/analytics/history/`)
- **Responsibility**: Manages persistence. `saved-reports/` handles templates created when clinicians pin certain prompts (matching Image 5), while `history-logger/` records an immutable audit log of generated reports (matching Image 6) to meet medical logging and compliance standards.

## Part 4: Separation of Concerns Diagram (Request Lifecycle)

```text
          │ (e.g., "Show pediatric vaccinations overdue")
          ▼
    [ FRONTEND ] ────────────────────────► App.tsx / Router
          │                                  │
          ▼ (HTTP Request with Session Token) ▼
    [ CONTROLLER ] ──────────────────────► promptController.ts (Receives request, delegates to Service)
          │
          ▼
    [ PRIVACY SERVICE ] ─────────────────► Redacts names/MRNs matching HIPAA rules via Presidio
          │
          ▼ (Safe Prompt)
    [ PROMPT PARSER ] ───────────────────► Gemini API parses intent (Outputs Structured JSON Schema)
          │
          ▼ (Parsed JSON Intent)
    [ QUERY BUILDER ] ───────────────────► Builds safe Prisma queries programmatically (No Direct AI SQL)
          │
          ▼
    [ REPOSITORY LAYER ] ────────────────► Performs secure SELECTs on PostgreSQL Database
          │
          ▼ (Raw Data rows)
    [ ANALYTICS ENGINE ] ────────────────► Computes KPIs, shapes chart arrays, filters cohort lists
          │
          ▼ (Anonymized Aggregate Metrics)
    [ SUMMARY SERVICE ] ─────────────────► Sends safe statistical outlines to Gemini for a written AI Summary
          │
          ▼ (Full Structured Payload)
    [ CONTROLLER ] ──────────────────────► Sends 200 OK Response
          │
          ▼
    [ FRONTEND RENDERING ] ──────────────► Recharts & Cohort Tables display beautiful dashboards (Image 6 & 7)
```
